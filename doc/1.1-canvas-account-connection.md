## 1. Problem Statement
A user cannot see any of their assignments until the app can reach their Canvas
instance on their behalf. Today there is a connection form on the landing page,
but the access token is collected before the user has an account and is not yet
stored securely or validated. Until a user can durably and safely link their
Canvas account, every downstream feature (sync, urgency sorting, notifications)
is dead.

## 2. Goals
- Let an authenticated user link exactly one Canvas account by providing a domain + access token.
- Validate the token against Canvas before saving it.
- Store the token encrypted, scoped to the owning user, never exposed to the client again.
- Let a user see connection status and disconnect/re-link at any time.

## 3. Non-Goals
- OAuth2 / Canvas Developer Key flow (token paste only for this story).
- Multiple linked Canvas accounts per user.
- Actually fetching assignments — that is story 1.2.

## 4. User Stories
- **As a student**, I want to connect my Canvas account with my school domain and a token so that the app can read my coursework.
- **As a student**, I want to know whether my connection is healthy so that I trust the data I see.
- **As a privacy-conscious user**, I want to disconnect and have my token deleted so that I stay in control.

## 5. Functional Requirements
- **FR-1.** The system MUST require an authenticated session before accepting a Canvas domain or token.
- **FR-2.** The system MUST validate the token by calling `GET /api/v1/users/self` on the supplied domain and rejecting the link if the call is not `200`.
- **FR-3.** The system MUST store the token encrypted at rest, associated with the authenticated `user_id`.
- **FR-4.** The system MUST NOT return the stored token to any client after saving (write-only from the client's perspective).
- **FR-5.** The system MUST allow a user to disconnect, which deletes the stored token and connection row.
- **FR-6.** The system SHOULD normalize the domain (strip scheme/trailing slash) and reject obviously invalid hosts before calling Canvas.
- **FR-7.** The system MAY surface the Canvas display name returned by validation as a friendly confirmation.

## 6. Non-Functional Requirements
- **Performance** — connection validation p95 < 2s (bounded by Canvas round-trip); 5s timeout with a clear error.
- **Security** — token encrypted with a server-side key from `CANVAS_TOKEN_ENC_KEY` (env var); decryption only in the backend sync path; HTTPS only; per-user authz on every route.
- **Privacy** — store only the token, domain, and Canvas user id. No assignment data stored by this story. Deleting the connection hard-deletes the row.
- **Accessibility** — form fields labelled, errors announced via `aria-live`, focus moves to the first invalid field. WCAG 2.1 AA.
- **Reliability** — validation is idempotent; re-linking overwrites the existing connection rather than duplicating it.
- **Observability** — log `canvas_connect_attempt` with outcome (`ok` / `invalid_token` / `unreachable`), user id, and domain — never the token.

## 7. Acceptance Criteria
- **AC-1.** *Given* a logged-in user with a valid token, *When* they submit domain + token, *Then* the connection saves, status shows "Connected" with their Canvas name, and the token is not present in any response body.
- **AC-2.** *Given* an invalid or revoked token, *When* they submit, *Then* validation fails with "We couldn't verify that token" and nothing is saved.
- **AC-3.** *Given* a connected user, *When* they click Disconnect, *Then* the row is deleted and status returns to "Not connected".
- **AC-4.** *Given* an unauthenticated request to any connection route, *When* it is made, *Then* it returns `401`.

## 8. Data Model
New table `canvas_connections`:

| Column | Type | Notes |
|---|---|---|
| `id` | uuid PK | |
| `user_id` | uuid FK → auth.users | unique (one connection per user) |
| `canvas_domain` | text | normalized host |
| `canvas_user_id` | bigint | from validation response |
| `token_ciphertext` | text | encrypted; never selected by client queries |
| `created_at` / `updated_at` | timestamptz | |

- Unique index on `user_id`.
- Row Level Security: a user may only read/write their own row; `token_ciphertext` excluded from any client-facing view.
- Migration: `supabase/migrations/002_canvas_connections.sql`.

## 9. API Surface
- `POST /api/canvas/connect` — auth required. Body `{ domain, token }`. Validates, encrypts, upserts. Returns `{ status, canvasName }` (no token).
- `GET /api/canvas/status` — auth required. Returns `{ connected: boolean, canvasName?, domain? }`.
- `DELETE /api/canvas/connect` — auth required. Deletes the connection.
- Rate limit: 5 connect attempts / user / minute.

## 10. UI / UX
- Move the existing landing-page connection panel **behind auth** into a Settings → Connections view.
- Flow: (1) signed-in user opens Connections, (2) enters domain + token, (3) sees inline validation spinner, (4) success → "Connected as {name}" with a Disconnect button.
- States: empty (not connected), loading (validating), error (bad token / unreachable), connected.
- Add a "How do I get my access token?" disclosure linking to Canvas account-settings steps.
- Respect ink + ember tokens; the token input uses a masked field. Mobile: single-column, full-width inputs.

## 14. Risks & Mitigations
| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Token leaks into logs | M | H | Central redaction; assert no token in log fields; review log statements |
| User pastes token on wrong domain | M | M | Validate domain reachability + `users/self` before saving |
| Encryption key rotation breaks decrypt | L | H | Store key id alongside ciphertext; document rotation runbook |

## 15. Rollout Plan
- Flag: `feature.canvas_connect` (default **off**).
- Sequence: migration → deploy backend routes → deploy Connections UI → flip flag for self → dogfood → GA.
- Rollback: flip flag off; `DELETE` route remains available so users can clear tokens.

## 16. Test Plan
- **Unit** — domain normalization, encrypt/decrypt round-trip, validation-response parsing.
- **Integration** — connect with mocked Canvas `200` / `401`; authz `401` on unauthenticated calls; disconnect deletes row.
- **E2E (Playwright)** — connect → status → disconnect happy path; bad-token error path.
- **Security** — confirm token never appears in any response or log; RLS denies cross-user reads.
- **Accessibility** — axe on Connections page; error focus/announcement check.

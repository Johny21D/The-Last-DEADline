## 1. Problem Statement
A connected user currently sees zero assignments: the sync path fetches from
Canvas but the `upsert_assignment` Postgres RPC fails silently, so nothing
lands in the database and the UI shows an empty list with no error. Sync is the
heart of the product — without reliable, observable ingestion of assignments,
every urgency, sorting, and notification feature has no data to act on.

## 2. Goals
- Fetch a user's active assignments from Canvas using their stored token.
- Upsert assignments into the database idempotently (re-running sync does not duplicate).
- Surface sync failures loudly — to logs and to the user — instead of silently returning 0.
- Record `last_synced_at` and a sync status the UI can show.

## 3. Non-Goals
- Real-time webhooks from Canvas (polling only this story).
- Computing urgency tiers / colors (that is a separate UI/logic story).
- Scheduled/automatic sync (this story is manual "Sync now"; cron is story 2.x).

## 4. User Stories
- **As a student**, I want my Canvas assignments pulled into the app so that I can see all my deadlines in one place.
- **As a student**, I want a clear error when sync fails so that I'm not staring at a silently empty list.
- **As a student**, I want re-syncing to update changed due dates without creating duplicates.

## 5. Functional Requirements
- **FR-1.** The system MUST decrypt the user's stored Canvas token server-side and never expose it to the client.
- **FR-2.** The system MUST page through Canvas assignments (Link-header pagination) until exhausted.
- **FR-3.** The system MUST upsert on a stable natural key (`user_id` + `canvas_assignment_id`) so re-sync is idempotent.
- **FR-4.** The RPC/path MUST return a structured result `{ inserted, updated, failed, error? }`; a failed upsert MUST NOT be swallowed.
- **FR-5.** The system MUST set `last_synced_at` and a `sync_status` (`ok` / `partial` / `failed`) on completion.
- **FR-6.** The system SHOULD skip assignments with no due date or store them as "no deadline" rather than erroring.
- **FR-7.** On Canvas `401`, the system MUST mark the connection as needing re-auth and tell the user.

## 6. Non-Functional Requirements
- **Performance** — sync of ≤200 assignments completes p95 < 8s; Canvas calls batched/paged, not N+1 per assignment.
- **Security** — token decrypted only in the sync worker; per-user authz; no token in logs.
- **Privacy** — store only fields the app uses (title, course, due date, html_url, points). No submission content.
- **Reliability** — idempotent upsert; partial failures recorded per-row, sync continues; retry with backoff on transient Canvas 5xx.
- **Observability** — **this is the fix for today's bug.** Log `sync_start`, per-page counts, and `sync_complete` with `{inserted, updated, failed}`. The RPC MUST raise/return Postgres errors to the caller and they MUST be logged with the failing row's key. No empty-success path.

## 7. Acceptance Criteria
- **AC-1.** *Given* a connected user with 12 Canvas assignments, *When* they trigger sync, *Then* 12 rows exist for that user and `sync_status = ok`.
- **AC-2.** *Given* a malformed assignment that fails to upsert, *When* sync runs, *Then* it is counted in `failed`, the error is logged with its key, other rows still persist, and `sync_status = partial`.
- **AC-3.** *Given* a revoked token, *When* sync runs, *Then* the user sees "Reconnect your Canvas account" and `sync_status = failed`.
- **AC-4.** *Given* a prior sync, *When* a due date changes in Canvas and sync re-runs, *Then* the existing row is updated (not duplicated).

## 8. Data Model

| Column | Type | Notes |

| `html_url` | text | deep link back to Canvas |
| `points_possible` | numeric null | |
| `synced_at` | timestamptz | |

- **Unique index `(user_id, canvas_assignment_id)`** — the upsert conflict target.
- Add `last_synced_at timestamptz` and `sync_status text` to `canvas_connections`.
- Migration: `supabase/migrations/003_assignments.sql`.
- `upsert_assignment(...)` RPC: `INSERT ... ON CONFLICT (user_id, canvas_assignment_id) DO UPDATE`, returning the row; **must surface SQLSTATE on error**, not swallow.

## 9. API Surface
- `POST /api/canvas/sync` — auth required. Triggers a sync for the caller. Returns `{ inserted, updated, failed, sync_status, last_synced_at }`.
- `GET /api/assignments` — auth required. Returns the caller's assignments (ordered by `due_at`).
- Rate limit: 1 manual sync / user / 30s.

## 10. UI / UX
- "Sync now" button on the dashboard showing last-synced time and a spinner during sync.
- States: never synced, syncing, synced-ok, synced-partial (banner: "Some assignments couldn't be imported"), failed (banner with Reconnect CTA).
- Empty list copy distinguishes "no assignments due" from "sync hasn't run / failed."
- Respect ink + ember; due dates rendered in monospace numerals.

## 14. Risks & Mitigations
| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| RPC fails silently (today's bug) | H | H | RPC returns/raises errors; caller logs `failed` count + SQLSTATE; AC-2 guards it |
| Conflict target mismatch → duplicates or no-op | M | H | Explicit unique index `(user_id, canvas_assignment_id)`; test re-sync idempotency |
| Canvas pagination missed → partial data | M | M | Follow Link `rel="next"` until absent; assert page count in logs |
| Token expired mid-sync | L | M | Detect `401`, mark connection, surface Reconnect |

## 15. Rollout Plan
- Flag: `feature.canvas_sync` (default **off**).
- Sequence: migration (table + RPC) → deploy sync worker → deploy "Sync now" UI → dogfood with own Canvas account → verify counts → GA.
- Rollback: flag off; assignments table retained (read-only) so prior data still displays.

## 16. Test Plan
- **Unit** — pagination follower, assignment-mapper (Canvas JSON → row), no-due-date handling.
- **Integration** — RPC upsert insert vs update; forced row failure → `partial`; `401` → `failed`. Run against a local Supabase with the real migration.
- **E2E (Playwright)** — connect → sync → list populated; revoke token → Reconnect banner.
- **Security** — token absent from logs and responses.
- **Manual** — sync own Canvas; confirm counts match Canvas UI.
